# Ride or Build decision guide

Use this file to decide whether to use an existing service/tool or build your own.

## Prefer Ride when

- The problem is generic.
- Existing tools are mature.
- Differentiation is not in this area.
- Maintenance cost would be high.
- Security/compliance requirements are satisfied by the vendor.

## Prefer Build when

- The workflow is core to your differentiation.
- You have unique data or domain-specific rules.
- Existing tools cannot fit the required workflow.
- Integration depth is strategically important.
- You need control over data, auditability, or deployment.
